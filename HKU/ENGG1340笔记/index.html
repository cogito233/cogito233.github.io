Linux & shell

Linux基本操作指令

目录名字 意义
./ 当前目录
~/ 用户目录，或者说是/home/{yourname}
../ 上一个目录
/ 根目录
/bin 二进制程序文件
/lib 库文件
/tmp 临时文件
/usr 用户程序文件
/dev 设备文件
  • Linux系统中很多东西都被赋予了文件的抽象。
指令名称 操作方法
ls 列出当前目录下文件 (-l 查看详细信息(例如：文件属性))
cd 切换目录
pwd 查看当前目录
sudo 以管理员权限执行
mkdir 创建一个目录
rm 配合-rf使用效果极佳，-r是目录，-f是无需逐一确认
mv 移动
apt Ubuntu专属包管理软件，
pip 这个有必要单独放在这里讲，py缺包了第一反应就是pip install xxx
cp 复制
cat/more/head 输出文件
echo 相当于print一个字符串，输出的是参数
read <variablename> 从标准输入流中读入
chown <filname> 777 修改一个文件的访问权限
which -a <ex-filename> 确定所有该名字可执行文件的路径

不常用但是很骚而且会考文件操作的指令

指令名字 用处
grep -E "" <filename> 匹配一个正则表达式
cut num1-num2 <filename> -b是以字节风格，-c以字符分割，-d是制表符分割
sort <filename> 按行字典序排序
spell <filename> 显示所有拼写错误
uniq <filename> 删除相邻的相同的行
wc <filename> 行数，单词数，字符数，文件名

正则表达式

  • 正则表达式从^开始，从$结束
  • [123]：123,[^123]：非123,[A-Z]：A到Z,[.]：非换行,[\s\S]：空白和非空白
  • 中括号后面根限定符，{3,15}：长度为3..15,{3,}:至少n次, *:0~n,+:1~n,?:0~1
  • .:单字符, |:或, ():子表达式

vi/vim

  • 千万不要用vim，如果不小心打开了vim请学会:wq
  • i是编辑模式，esc是命令模式

表达式

参数传递

可以理解为一组保留的变量名(环境变量?)代表当前的运行环境

参数名 意义
$0 执行的文件名，包括文件路径
${number} 第n个参数
$# 传递到脚本的参数个数
$* 一个字符串显示所有参数
$@ 一个数组显示所有参数
$- 当前选项（不知道有什么用）
$? 最后命令的退出状态
$$ 当前进程的ID号
$! 最后一个进程的ID号

  • 用于判断的表达式放在[]里即可
  • 算数比较：-eq/ne/gt/ls/ge/le

Git

适用于没有GitHub Desktop的远程服务器上

GitHub Desktop是yyds

仓库管理

基本命令 效果
git init 在本地创建一个git的repo
git add readme.txt 将一个文件加入repo
git commit -m "comment" 将文件提交到仓库
git status 查看仓库当前状态
git diff readme.txt 查看文件被怎么修改了(当前是未commit状态)
git checkout -- readme.txt 丢弃文件在工作区的修改
git reset HEAD <file> 丢弃文件在暂存区的修改
git remote add origin <url> 添加远程库
git push -u origin master 推送远程库
git clone <url> 克隆远程库

版本控制

HEAD指针指向当前版本

HEAD^指针指向上一个版本

HEAD~100指针指上100个版本

1094a...指向对应commit id的版本

基本指令 效果
git log --pretty=oneline 显示日志信息（不加--pretty=oneline则为详细信息）
git reset --hard HEAD^ 把版本重置到对应指针的版本

参考内容

分支管理

不要随便切换分支，你会失去工作区和暂存区修改了的东西

基本指令 效果
git stash 将目前工作区和暂存区的东西存下来以后再用
git stash pop 恢复原先状态
git stash list 列出缓存的栈
git stash apply stash@{0} 回到指定的缓存
git branch 查看分支
git branch <name> 创建分支
git checkout/switch <name> 切换分支，checkout -b或switch -c代表创建+切换
git merge <name> 合并某分支到当前分支
git branch -d <name> 删除某分支
git checkout -b dev origin/dev 切换到远程库的dev分支

远程仓库

基本指令 效果
git push origin <branch-name> 更新相应的branch
git clone <url> 克隆对应的仓库
git checkout -b dev origin/dev 切换到devbranch
git pull 从远程仓库更新
git branch --set-upstream-to <branch-name> origin/<branch-name> 在本地和远程仓库间的两个branch间建立联系

编译选项 意义
-O2 O2优化开关

MakeFile

语法格式是.sh

举一个小栗子

对于每个需要make的文件，管理其依赖关系。

第二行<tab>后面为该文件的编译指令

标识符 意义
$@ 目标文件
$< 依赖列表最左边文件
$^ 全部依赖列表文件
.PHONY: 用来虚假执行的假目标

CMake

变量名 含义
${PROJECT_SOURCE_DIR} 当前工程源代码的最上层目录
${PROJECT_BINARY_DIR} 当前工程编译后文件的最上层目录
函数名 用法
project(<project-name>) 声明项目名字
add_subdirectory(<directory-name>) 添加库和可执行文件子目录
add_executable(<execute-name> ${DIR_SRCS}) 将一些源码合并编译为可执行文件
add_library(<lib-name> STATIC/SHARED <program-name>) 生成静态/动态库(默认是静态)
target_link_libraries (<execute-name> ${EXTRA_LIBS}) 给某个可执行文件link上lib
aux_source_directory(<dir> <variable-name>) 找出所有的源程序文件并存在变量里
set(<variable-name> <value>) 改变变量值, 也可以自加
include_directories("<dir>") 添加头文件搜索路径
configure_file(<file1>,<file2>) 将<file1>拷贝到<file2>
option(<option-name> "text" ON) 打开某个开关，可以在程序里#ifdef

项目编译相关

  • 如果使用Makefile，在Makefile所在目录直接启动终端执行make <execute-name>即可
  • 使用CMake则可以在根目录下执行cmake .
